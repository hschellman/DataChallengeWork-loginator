LArWrapper¶

class LArWrapper.LArWrapper(debug=False, fcl=None, flist='', o='temp*.root', n=None, nskip=0, appFamily=None, appName=None, appVersion=None, projectID=None, sam_web_uri=None, processID=None, dataTier='sam-user', dataStream='test', deliveryMethod=None, workflowMethod=None, formatString='runLar_%s_%s_%%tc_%s_%s_%s.root', processHASH=None, replicas=None)[source]¶

LArWrapper - generic wrapper for LArSoft that works with samweb and dd

Many of the options are the same as those for lar

$lar -h #for more examples
Parameters

  • debug (bool) – add printout and send stdout to console instead of a files

  • fcl – LAr option - fcl file to use

  • flist – LAr/dd option - list of space delimited files to process from dd

  • appFamily (str) – LAr option - Optional Information about process

  • appVersion (str) – LAr option - Optional Information about process

  • appName (str) – LAr option - Optional Information about process

  • projectID (int) – LAr option - DD or samweb project ID

  • sam_web_uri (str) – LAr/samweb option - samweb project url

  • processID (int) – LAr/samweb option - samweb process ID

  • processHASH (str) – dd option - process ID from dd

  • dataTier (str) – LAr option - data tier for output

  • dataStream (str) – LAr option - data strem for output

  • deliveryMethod (str) – required parameter to tell samweb from dd/wfs

  • workflowMethod (str) – optional parameter - batch/interactive/wfs

  • formatString (str) – optional format the output log names

Returns

the lar return code

Return type

int
